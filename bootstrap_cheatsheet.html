Bootstrap One-Page Cheat Sheet

A fast reference for what Bootstrap is, how its grid works, which classes matter most, and when to use it in real projects.

What it is Front-end framework for responsive layout and UI.
Main pieces Grid, components, utilities, breakpoints.
Best for Prototypes, dashboards, MVPs, internal tools.
Remember Bootstrap speeds up styling, but custom CSS is still allowed.

1. Core Idea

Bootstrap gives you a ready-made system for building pages faster. Instead of styling everything from scratch, you use built-in patterns.

Grid Components Utilities Responsive Design
Memory formula: Bootstrap = grid + components + utilities

2. Layout: Container, Row, Column

<div class="container"> <div class="row"> <div class="col-12 col-md-8">Main</div> <div class="col-12 col-md-4">Sidebar</div> </div> </div>
  • container: centers and pads page content
  • row: creates a horizontal grid row
  • col-*: defines responsive column width
  • col-12: full width on small screens
  • col-md-8: 8/12 width from medium screens and up

6. Bootstrap vs CSS vs JavaScript

Bootstrap Styling system and UI toolkit for layout, components, and responsiveness.
Plain CSS You write the design system yourself for maximum control.
JavaScript Programming language for logic and interactivity, not a styling framework.

7. Getting Started

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet" >
  • Add Bootstrap CSS first.
  • Use Bootstrap classes in regular HTML.
  • Add Bootstrap JavaScript only for interactive components when needed.

8. Best Workflow

1. Start with page structure using container, row, and col-*.
2. Add components like buttons, cards, navbars, and form controls.
3. Use utility classes for spacing, alignment, colors, and display.
4. Add custom CSS only when Bootstrap defaults are not enough.

9. Good Choice When...

Use Bootstrap for:
prototypes, dashboards, MVPs, and teams that need a consistent UI quickly.
Less ideal for:
highly custom brand-heavy designs or projects that require a unique design system.

10. Important Concepts

  • You still write HTML when using Bootstrap.
  • Bootstrap reduces CSS work, but does not remove the need for design decisions.
  • Responsive behavior is one of Bootstrap’s biggest strengths.
  • Bootstrap is mostly CSS, with optional JavaScript for selected interactive components.
  • Too many utility classes can make HTML harder to read, so use them thoughtfully.
